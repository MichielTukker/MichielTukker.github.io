My First Post

April 26, 2020 2-minute read

Welcome to my new website

This is the first post on my new website using Hugo. Everything appears to work now. I’ve set this up as a static site, so I don’t have to worry about the latest php exploit or other vulnerabilities.

How it works

The site is powered by Hugo, a static site generator. The content of the site is written in markdown and stored in a git repository. On my local machien I can write pages and content and test on a local instance of Hugo. When I’m satisfied with the content I can publish the site.

I have a bare repository on this server, where I can push the commits/changes from my local git repository to my server using git push production master. The repository on my server is configured as a remote repo via ssh. This bare repository has a git post-receive hook, which checks the commit, pulls the changes into a local copy of the git repository and then runs Hugo to publish the website. The website is exported into the html directory of my webserver.

I got the idea from this Blog post describing the general idea, but that doesn’t work because I’m using a Hugo theme via git submodules (as the Huge quickstart guide tells you to!). Appearantly you can’t init and/or update submodules in a bare repository and git submodules won’t listen to the --work-tree and --git-dir flags. You can’t push changes to a non-bare repository, because Git doesn’t like that. That’s why I took the additional step of having a local git repository on the server where I can pull the changes from the bare git repository (I got the idea from this StackOverflow answer).

It works!
