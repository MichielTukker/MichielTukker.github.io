Switching to Fedora 32

So I switched to Fedora 32 last week, after getting annoyed with Ubuntu 20.04 pushing snap-packages everywhere. Fedora has been a delight, although I’ve had several small hickups. I don’t blame Fedora for this, the problem is that I’ve grown accustomed to the ease of use of Ubuntu or Windows at the cost of using non-free software. Fedora ships only open source / free software in their standard package repositories which I think is a good thing. I though I’d document my hickups here, it may be of use to someone (probably myself during a future re-install of my pc)

My “issues” sofar are:

  • Guake terminal doesn’t work on Wayland, results in all kind of wierd placement and graphical issues
  • Docker doesn’t work, because Fedora uses cgroups v2, whereas docker doesn’t understand that (yet).
  • Openfoam on Ubuntu can be installed via apt, but on fedora it’s not available via dnf.
  • no native spotify client in the package repository (but workarounds available)
  • Nvidia driver not installed by default. (but available from rpm-fusion repository)

Guake terminal

I really like Guake terminal for it’s quick access and flexibility. However, it didn’t work for me in Fedora 32. I seemed to have both this and this issue. Guake terminal doesn’t seem to work on Wayland, which is the default display server on Fedora 32. Now, I don’t mind Wayland and it worked really well for the few days that I used it, but I really, really like Guake terminal. That basically means that I’m stuck with X.org for the forseeable future.

Docker on fedora 32

Docker doens’t work out of the box on Fedora 32, because Fedora uses cgroups v2 by default and Docker doesn’t support this. This article does a great job of explaining the benefits of cgroups v2. This page nicely explains the various options that you have to get docker working on Fedora 32. As I preferred to used the standard Moby-engine, I switch back to using cgroups v1, until support for cgroups v2 is included in moby-engine. you do this by running:

sudo dnf install grubby
sudo grubby --update-kernel=ALL --args="systemd.unified_cgroup_hierarchy=0"

Openfoam on Fedora

I want to use Openfoam via the excellent Freecad plugin, CfdOf. However, that plugin assumes that Openfoam is installed on the local file system, like /opt/openfoam. Fedora doesn’t include a package for Openfoam in their repositories, and Openfoam only provides a docker image for running on linux distributions other than Debian/Ubuntu. I don’t have a real solution for this, but it appears that someone is already working on this: https://forum.freecadweb.org/viewtopic.php?p=357926

Spotify

As fedora only ships free software by default, spotify is not included in the default repositories. This website lists three options.

Nvidia driver

Fedora ships with the open source nouvea driver for Nvidia cards by default. While this works perfectly fine for normal desktop user, I’d like to use CUDA which is not possible with Nouveau. Ubuntu allows the user to install the propietary Nvidia driver during the installation process, but Fedora doesn’t ship non-free drivers. However, there’s a RPM Fusion repository with the non-free Nvidia driver available.
